Inventory Integrity Audit

A professional-grade SQL solution for tracking warehouse stock levels and discrepancies — JOINs for validation, triggers for automated audit trails, and queries built for real reconciliation workflows.

SQL INNER JOIN Triggers CHECK Constraints GROUP BY

Project overview

Born from 20 years of watching physical counts disagree with WMS records, this project models the exact pattern operations teams need: a system-of-record table, a physical audit log, automated change tracking, and reporting queries that surface variance before it becomes a shipment error.

Architecture

Inventory table

System of record for part numbers and quantities with CHECK constraints enforcing data integrity at insert time.

Audit table

Captures physical counts to identify mismatches between system records and what is actually on the shelf.

Trigger automation

Logs every inventory change automatically — no manual audit entry, no gaps in the trail when shifts turn over.

Key queries

The system uses a JOIN to compare system records against manual counts, while the trigger ensures an audit trail exists without requiring manual log entry.

  • Audit report — highlights items where system vs. actual counts don't match
  • Hot list — identifies the top 5 parts with the most frequent audit mismatches
  • Discrepancy analysis — calculates variance between expected and physical quantities
